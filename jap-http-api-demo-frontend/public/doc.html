Bearer 第三方登录测试

对 Bearer 配置项的说明 我们知道 Bearer 认证其实就是在 Http 请求中携带一个 Token 字符串来实现的。但是这个字符串从哪儿来呢，需要我们携带用户信息向第三方服务器的指定地址发起认证请求来获得。对应 Bearer 示例流程图的第三步：

所以在进行 Bearer 认证时，我们需要指定
  1. Bearer Token 的签发请求地址
  2. Bearer 认证预请求方式：这个是用来指定在进行 Token 签发时用户信息如何发送到第三方，你可以看到，有 by_header（将用户信息放到请求头中）、by_body、by_params 这些传递方式， 还有两个比较特别的传递方式 by_basic、by_digest，这是因为在进行 Token 签发时我们可以使用 BASIC 或者 DIGEST 认证。

第三方 Bearer Token 签发请求地址： http://localhost:8088/auth/token
Bearer 认证预请求方式： by_basic
第三方登陆地址：http://localhost:8088/api/v1/source1
第三方HTTP请求方式： GET
用户名：admin
密码： admin
